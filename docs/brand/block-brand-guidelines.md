# Just Build with Blocks Brand Guidelines

Date: 2026-03-06

## Positioning

`Just Build with Blocks` should feel like an OpenAI product that unexpectedly became a premium custom block-set generator.

- OpenAI owns the shell.
- `BLOCKS` owns the wrapper.
- The hero object owns the credibility.

The target feeling is premium studded construction-toy confidence without direct brand use or copied retail systems.

## Product promise

- Upload a logo image or describe a brand or idea.
- Reveal an original `BLOCKS` set box.
- Step into a guided 3D builder studio.
- Leave with instructions, manifests, real exports, and a film-ready keepsake set.

## Visual system

### OpenAI shell

- OpenAI Sans everywhere
- white, graphite, and restrained blue
- calm hierarchy
- generous spacing
- product confidence, not mood-board styling

### `BLOCKS` wrapper

- big blue packaging field
- original red `BLOCKS` badge
- bright yellow accents
- playful metadata rails and badges
- glossy toy-native energy

### Hero object

- mostly monochrome
- disciplined geometry
- buildable silhouette first
- collectible shadows and lighting
- small base, plinth, or support details are allowed when they improve credibility

## Instruction language

The instruction system should feel immediately understandable and toy-native while staying original.

Use:

- pale sky-blue page fields
- large step numerals
- parts-needed callouts
- arrows and rotation indicators
- clean subassembly grouping
- strong whitespace
- readable pacing

Avoid:

- copied covers
- copied numbering grammar
- warning layouts that mirror retail manuals
- one-to-one page architecture from known brands

## Legal-safe guardrails

The emotional target is familiarity, not counterfeit mimicry.

Do not ship:

- official construction-toy brand logos, names, or wordmarks
- copied retail set-box layouts
- source-identical typefaces or numbering systems
- direct references to known minifigure systems
- instruction-book replicas

Do ship:

- an original `BLOCKS` badge
- original metadata labels such as `Builder age`, `Build ID`, `Pieces`, and `Mode`
- original cover composition
- original instruction language
- an original studio UI

## Tokens

### Color roles

- `Studio White` `#FFFFFF`
- `Graphite Ink` `#101828`
- `Signal Red` `#C4281C`
- `Orbit Blue` `#0055BF`
- `Bright Yellow` `#FFD500`
- `Builder Green` `#00C853`

### Typography

- Display: `OpenAI Sans`
- UI: `OpenAI Sans`
- Labels and counts: `OpenAI Sans`
- No secondary display face

### Shape language

- Shell: disciplined rounded rectangles
- Wrapper: chunkier rounded forms with clear block rhythm
- Object: clean, believable, and physically grounded
- Controls: slightly blocky, never retro-game literal
- Surfaces: glossy, tactile, layered, and obviously physical

## Copy voice

- warm
- direct
- short
- ambitious
- playful without sounding childish

Preferred lines:

- Upload it. Or describe it. Build the set.
- Start with a mark. End with a set.
- The software is the board. The blocks are the possibilities.

## Motion rules

- snap
- slide
- settle
- click
- reveal

Motion should bridge the wrapper and the object. It should make the product feel precise and tactile, not loud.

Avoid:

- liquid motion
- goo
- hologram tropes
- parody nostalgia
- overblown trailer behavior

## Product bar

Every sprint point should pass this gut check:

- Does the first frame read as a product, not a concept board?
- Does the shell still feel recognizably OpenAI?
- Does the wrapper feel original and shelf-ready?
- Does the object look buildable enough to believe?
- Would someone want to send a screenshot before we explain anything?
