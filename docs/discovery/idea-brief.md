# Idea Brief: Just Build with Blocks Signature Product

Date: 2026-03-06
Repository: Just-Build-Blocks
Working stack: React, Vite, TypeScript, Vitest

## 1) Problem

People can imagine a great block-set version of a brand or idea, but most tools stop at mockups, mood boards, or loose concept art. This project closes that gap. It should let someone start from a logo image, brand name, or short prompt and leave with a premium block-set artifact that feels buildable, branded, and complete.

## 2) Target User

Primary user: creative builders, brand teams, and internal reviewers who want a product-grade set concept rather than a loose visual experiment.
Secondary user: the OpenAI/Codex team, who need a polished flagship artifact that feels ready for screenshots, demos, and film.
Out of scope: open-ended sandbox creators, retail checkout flows, or counterfeit use of official toy-brand assets.

## 3) Outcome

The user uploads a logo or enters a brand name or prompt and gets:

- a buildable premium block-set concept
- an original set-box presentation
- a guided 3D studio view
- an original instruction-book style artifact
- a real export bundle with model files and manifests
- a Remotion 3D film with sound

## 4) Success Signals

- Leading indicator: a first-time user can reach a product-looking set-box frame from image or prompt input without needing expert settings.
- Lagging indicator: the generated object, exports, and film are strong enough to share as a flagship Codex artifact rather than a rough proof.
- Manual acceptance check: a recognizable logo or named brand becomes a buildable set with plausible structure, synchronized instructions, downloadable exports, and a sound-led film path.

## 5) Constraints

- Technical: the system should stay browser-first at the app layer, but the core must move toward a deterministic set engine with stable exports and Remotion-ready scene data.
- Product: the experience must feel complete, not stitched together from unrelated demo surfaces.
- Brand/legal: the work should feel instantly familiar to anyone who knows iconic studded block toys, while never depending on official names, logos, layouts, or direct mimicry.

## 6) Non-Goals

- This effort will not become a freeform brick-by-brick CAD editor.
- This effort will not promise manufacturing-grade sourcing, pricing, or retail availability in this phase.
- This effort will not chase many different set genres before the flagship collectible path is excellent.

## 7) Assumptions

- Assumption 1: a disciplined hero object with a premium wrapper is more credible than a larger but less buildable world.
- Assumption 2: image input and prompt input should converge on the same canonical set pipeline.
- Assumption 3: the Remotion film should prove the same model and instruction phases, not invent a separate story layer disconnected from the build.

## 8) Open Questions

- Question 1: what minimum geometry and support fidelity is required before we call the output buildable with confidence?
- Question 2: which parts of the 3D studio should be live and interactive versus rendered or implied through Remotion?
- Question 3: how much instruction-page layout should be deterministic versus art-directed per flagship release?
