# Contributing

## Branches

- Use `main` for the shared trunk.
- Use `codex/sprint-XX-slug` for sprint branches.

## Sprint rhythm

1. Open the branch early.
2. Keep commits small and narrative.
3. Do not merge until the sprint point is demoable.
4. Attach at least one screenshot or GIF to the PR.
5. Tag each merged sprint point.

## Required checks

- `npm run lint`
- `npm test`
- `npm run build`

## Product bar

- The shell should feel OpenAI-clean.
- The stage should feel unmistakably like a custom block-set product.
- Public artifacts must feel close in spirit, not counterfeit in execution.
